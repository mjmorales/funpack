WHY A NAIVE PER-VERSION FREE CORRUPTS STATE (load-bearing constraint). The version model uses STRUCTURAL SHARING with a row-level alias FINER than commit_version's table-level sharing:
- commit_version (state.odin:152) shares UNCHANGED Version_Tables by reference (table-level COW).
- new_tick_tables (tick.odin:1359) seeds each working row via append(rows,row) — copies the Row STRUCT but Row.fields is an Odin map (reference type), so an UNWRITTEN working row's blackboard map HEADER aliases the prior version's map backing + its Record/List/String/Variant payloads.
- write_blackboard (tick.odin:1227) allocates a FRESH map (+clones structural payloads via value_to_field_value→clone_record/list, interp.odin:1131) ONLY for a row the tick actually wrote.
- commit_tick_tables (tick.odin:1404) shallow copy(rows,working) → the committed NEXT version's unwritten rows share the PRIOR version's fields map + payloads by reference.
RESULT: freeing version N's allocator frees the fields maps/payloads of every row N held that N+1 did not rewrite; N+1 still points at them → use-after-free. A per-tick free_all is strictly worse.
